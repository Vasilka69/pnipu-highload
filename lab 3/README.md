# Отчет по лабораторной работе 3

Целью лабораторной работы была реализация простой очереди задач на базе Redis.  
В проекте собран сервис, который умеет добавлять сообщения в очередь как по одному, так и пакетно.  
Для генерации неравномерной нагрузки предусмотрены два режима: создание набора тестовых сообщений с случайной задержкой и загрузка задач из файла `queue-input.txt` с той же моделью задержек.

Обработка очереди вынесена в отдельный потребитель, который с заданным интервалом забирает элемент из Redis, делает искусственную паузу на обработку и переносит задачу в список обработанных сообщений.  
Дополнительно реализованы методы для просмотра статистики по очереди и истории уже обработанных элементов.  
За счет этого можно наблюдать накопление задач, их постепенное выполнение и текущее состояние очереди в любой момент времени.

В результате получился наглядный пример асинхронной обработки данных.  
Производитель и потребитель работают независимо друг от друга, поэтому всплески входящих сообщений не блокируют приложение напрямую, а сглаживаются за счет очереди.  
Такая схема хорошо показывает, как Redis можно использовать не только как кэш или key-value хранилище, но и как простой механизм распределения фоновых задач.
