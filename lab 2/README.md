# Отчет по лабораторной работе 2

В этой лабораторной работе приложение из первой части было доработано так, чтобы состояние больше не хранилось внутри отдельного экземпляра сервиса.  
Для этого подключен Redis, а счетчик запросов вынесен в общее хранилище.  
За счет этого при обращении к разным экземплярам приложения возвращается одно и то же актуальное значение счетчика, а локальные значения экземпляров.

Кроме общего счетчика был реализован отдельный REST API для таблицы лидеров на основе Redis Sorted Set.  
Через него можно записывать очки игрока, получать верхнюю часть рейтинга и определять место конкретного игрока.  
Таким образом в работе были использованы сразу два типовых сценария применения Redis: хранение общего состояния между экземплярами сервиса и работа со структурой данных, оптимизированной под ранжирование.

Итог лабораторной работы состоит в том, что приложение стало ближе к реальной распределенной схеме.  
Несколько контейнеров сервиса могут работать параллельно, но все они используют единое быстрое хранилище.  
В результате общий счетчик и рейтинг игроков остаются согласованными независимо от того, какой экземпляр обработал конкретный запрос.
