# Отчет по лабораторной работе 1

В рамках лабораторной работы было собрано Spring Boot приложение с HTTP endpoint `/counter`.  
При каждом обращении сервис увеличивает локальный счетчик и возвращает текущее значение вместе с информацией об экземпляре приложения.  
Для проверки горизонтального масштабирования подготовлен запуск трех одинаковых экземпляров в Docker на разных портах. 

Для распределения запросов настроен Nginx.  
В конфигурации предусмотрен базовый режим `round robin`, а также оставлены варианты для переключения на `least_conn`, `random`, `least_time`, `hash` и `ip_hash`.  
Дополнительно настроены параметры `max_fails` и `fail_timeout`, чтобы можно было проверить реакцию балансировщика на отказ одного из backend-сервисов.  
В ответы прокидываются служебные заголовки `X-Upstream-*`, поэтому удобно видеть, какой узел обработал запрос и как отработал upstream.

В результате получен рабочий стенд для демонстрации балансировки нагрузки и базовой отказоустойчивости.  
При обращении через Nginx запросы распределяются между несколькими экземплярами приложения, а при отключении одного из них его можно исключить из обработки до восстановления.  
Лабораторная показывает разницу между одиночным сервисом и масштабируемой схемой с балансировщиком перед группой backend-узлов.
